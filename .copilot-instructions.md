# Task Manager Application - Copilot Instructions

You are assisting with development of a **full-stack Task Manager application** with ReactTypeScript frontend and Express.js backend.

## Project Structure

- **`backend/`** - Express.js REST API server (port 5000)
- **`frontend/`** - React + TypeScript SPA (port 3000)
- Each has its own `package.json` and dependencies

## Technology Stack

- **Frontend**: React 18, TypeScript, Axios, CSS
- **Backend**: Express.js, Node.js, UUID, CORS
- **Data Model**:
  ```typescript
  Task {
    id: string (UUID)
    title: string (required)
    description: string (optional)
    completed: boolean
  }
  ```

## Code Conventions

### TypeScript
- Use functional components with React hooks for all new components
- Use TypeScript interfaces for all props and state types
- Example: `const MyComponent: React.FC<Props> = ({ prop }) => { ... }`
- Import statements at top of file, organized (React first, then libraries, then local)

### Component Structure
- Components live in `frontend/src/components/`
- One component per file
- File names match component names (PascalCase)
- Props interface defined before component

### Styling
- All styles in `frontend/src/App.css` using CSS utility approach
- Use CSS variables for colors (defined at :root)
- Mobile-first responsive design with @media queries
- BEM-style class naming Convention: `.block`, `.block__element`, `.block--modifier`
- Examples: `.task-item`, `.task-item.completed`, `.task-item__content`

### Backend (Express)
- All routes in `server.js` with comments grouping related endpoints
- Middleware at top: cors, express.json()
- In-memory array `let tasks = []` for data storage
- Use express.json() for request bodies
- Always validate required fields (title required for tasks)
- Return appropriate HTTP status codes (201 for POST, 404 for not found, 400 for bad request)
- Error handling middleware at bottom

### API Conventions
- Base path: `/api/tasks`
- Request/response format: JSON
- RESTful principles (GET, POST, PUT, DELETE)
- Consistent error response format: `{ error: "message" }`

## Common Tasks

### Adding a New Feature
1. If it's UI: Create new component in `frontend/src/components/`
2. If it affects data: Update Task interface and add API endpoint
3. Add component to App.tsx if it's a top-level feature
4. Export component from file, import in App.tsx

### Making API Calls
Use axios (already imported in App.tsx):
```typescript
try {
  const response = await axios.get(`${API_URL}/${id}`);
  // Handle response
} catch (err) {
  setError('User-friendly message');
}
```

### Styling New Elements
1. Add class to JSX element
2. Add CSS rules to App.css following conventions
3. Use existing CSS variables for colors
4. Test responsiveness at 768px breakpoint

## Backend Routes Reference

- `GET /api/tasks` - Returns all tasks array
- `POST /api/tasks` - Requires `{title, description?}`, returns new task
- `PUT /api/tasks/:id` - Partial updates, returns updated task
- `DELETE /api/tasks/:id` - Returns deleted task
- `GET /health` - Server health check

## Development Workflow

1. **Backend changes**: Modify `backend/server.js`, server auto-reloads with nodemon
2. **Frontend changes**: Modify TypeScript/CSS, app auto-reloads with react-scripts
3. **Test both**: Backend on port 5000, Frontend on port 3000
4. **Debugging**: Use browser DevTools for frontend, console.log/node inspect for backend

## DO's and DON'Ts

✅ **DO**
- Use TypeScript types for all props and state
- Handle errors gracefully with user-friendly messages
- Keep functions small and focused
- Comment complex logic
- Test API connectivity before making calls

❌ **DON'T**
- Use `any` type in TypeScript (use proper types or interfaces)
- Make synchronous API calls
- Hardcode values (use constants/env variables)
- Modify tasks array directly on backend (use splice/map pattern)
- Skip error handling on API calls

## Useful Patterns

### Error Handling
```typescript
const [error, setError] = useState<string | null>(null);

try {
  // API call
  setError(null);
} catch (err) {
  setError('Failed to complete action');
  console.error(err);
}
```

### Async Fetching
```typescript
useEffect(() => {
  const loadData = async () => {
    try {
      // fetch data
    } catch (err) {
      // handle error
    }
  };
  loadData();
}, [dependencies]);
```
